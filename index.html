A Bit About HTML and CSS
Lesson 1: Basics of the Web

The World Wide Web is a collection of HTML documents that was invented in the 1990s and now encompasses around 30 billion pages from around the world.

All sorts of files can be found on the web, such as text, images, videos and music.

You and the server

A computer requests data from a remote server using HTTP, the main protocol of the web. A web browser then interprets and displays the data on a screen for the user (that's you!) to see.

HTML

HTML stands for Hypertext Markup Language and is the heart of the web. HTML documents form the majority of the content on the web, including text, images, videos and music. HTML documents contain text content which describes "what you see" and markup which describes "how it looks".

Tags and Elements

HTML documents are made of elements, which include a set of tags and content within those tags. HTML elements can be either inline or block.

<tag_name> contents </tag_name>

One thing to remember is that computers are stupid. They only understand what a programmer explicitly tells them to do. For example, if there is no closing tag on an element, the last command will continue to be valid until the computer sees a corresponding closing tag.

An HTML tag can have multiple attributes, or additional descriptors.

<tag_name attr="value"> contents </tag_name>

The <a> tag, or anchor tag, has an attribute of 'href' followed by a value of the attribute, which in this case, is the URL of a link.

<a href="URL"> contents </a>

Similarly, the <img> tag has an attribute of 'src', with the value being the name of the image file. In addition to the image source, it's often recommended to also include an 'alt' attribute with the value being alternative text that is displayed when the image won't load. Note that the <img> tag does not contain any content, and so has no closing tag. The <img> tag is considered a "void tag".

<img src="image.jpg" alt="alternate text">

Another void tag is the <br> tag; it is merely a line break, without any content associated with it. It's used to separate lines of text, since in HTML, all whitespace is rendered as a single space.

Block elements form an "invisible box" around the content inside of them. Examples include:

  • <p> : paragraph
  • <div> : section division
Inline tags include:
  • <br> : break, or new line
  • <em> : italics
  • <b> : bold
  • <img> : image
  • <span> : often used to add styling
  • and others.
Lesson 2: Creating a Structured Document with HTML

If a website is seen as a structure, or say, a house, then:

  • HTML is the structure: the walls, the room divisions, where each room is located.
  • CSS is the style: wall color, curtains, decorations, carpeting.
  • Javascript is the interactive component: the TV, the garage door opener.

HTML and CSS are both languages, with their own specific rules and syntax, just as the English language has grammatical rules. A DOM [Document Object Model] is a convention for representing and interacting with HTML elements.

Because each element in a page is rectangular, a design can be broken down into a series of nesting boxes. Each box can be identified by a <div> tag, and content can be added within the opening and closing tags. The nesting of the boxes creates the tree-like structure of HTML, called the DOM tree. A nested element is indented from the parent element above. The more indented the element, the more nested the box.

CSS, which stands for Cascading Style Sheets, determine the look of the website, including position, color, font, and margins. Each box, or each <div> element, can be styled using individual class attributes, sort of like tags on boxes, to help you remember what those boxes are for. Just as you would label a physical box with a meaningful name, class attributes should also have meaningful names.

Lesson 3: Adding Style to Structure with CSS

Don't Repeat Yourself

When writing CSS, you may realize that you want several of the same type of elements to have the same look. You can give each element their own individual styling, and repeat the same style for each element, possibly with minor adjustments. If you ever want to make a change to all elements, you would have to go through and individually change each style. Having many repetitions like this in your code can often lead to user error and typographical mistakes.

By establish unique class attributes for each type of element, you would only need to change the style for that class, and those changes will be reflected in all the elements that use the specified class. This can save both time and trouble.

CSS

As mentioned before, CSS stands for Cascading Style Sheets. The "style sheets" part is easy to understand; it's the "cascading" part that can cause some confusion.

When styles are cascaded, the most specific rule is applied to every element. The program will continue traveling down the cascade of elements and rules until it come to one that best describes the element; that becomes the rule that is applied to the element.

A selector is what determines the element a rule is applied to. For instance, to apply a style to a class, use 

.classname {attribute: value;}

There are 3 different ways to apply CSS to a webpage:

  1. Add <style> elements to the <head> section of the website.
  2. Link a separate .css file to the html file with a <link> element in the <head> element of your website.
    <link rel="stylesheet" href="main.css">
  3. Write the styling inline into the HTML. This can be useful in certain cases, but should be used carefully to avoid unnecessary repetition.

Comments

Comments are lines of text that the computer will ignore. They are used as notes to help explain sections of code, or as reminders to users who read the code long after it's been written.

HTML comments begin with <!-- and end with -->.

CSS comments begin with /* and end with */.

HTML Boxes and Positioning

All HTML elements are rectangular, or boxes, and each of those boxes have 4 components: the content, the padding, the border and the margin. Each of these components can be adjusted individually using CSS, and it can get confusing when trying to determine the final size of an element. Two techniques to help with sizing issues are:

  1. Use percents instead of pixels when sizing boxes. This will size the boxes relatively vs setting an absolute size.
  2. Set the box-sizing attribute to border-box for every element. In order to ensure this command works on all browsers, be sure to add browser specific prefixes: 
    *{
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    -ms-box-sizing: border-box;
    box-sizing: border-box;
}

Since <div> tags are block elements, by default, they take up the entire width of the website. To override this behavior, use display: flex; so that <div> sections can appear next to each other.

Coding and Testing

To test changes in CSS, you can open Developer Tools in the browser and adjust the CSS code on the fly for each element you see on the page. These changes are temporary and are only meant to give a preview of what the change will look like. Once you are happy with the styling, you can go back to your CSS code and make the changes permanent.
